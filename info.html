Finite Automata Visualization

What is this Project About

This project is a web-based Finite Automata Visualization tool where you can:

  • will have 2 modes(buttons and manual code)
  • Build DFAs and NFAs
  • Simulate them on an input word stepwise

What are Finite Automata?

Finite automata are abstract computational models used to design and analyze the behavior of digital systems. They consist of a finite number of states and transitions between those states based on input symbols.

Deterministic Finite Automata (DFA)

A DFA is an automaton where each state has exactly one transition for each input symbol. This determinism makes DFAs predictable and efficient, which is why they are widely used in software applications, such as lexical analysis and pattern matching.

Nondeterministic Finite Automata (NFA)

An NFA can have multiple transitions for the same input symbol or even transitions without any input (ε-transitions). This flexibility simplifies the design of automata for complex patterns, though NFAs are often converted to DFAs for execution.

How to Use this Project

This project has two modes:

Create Mode
Build a DFA or NFA by adding states and arrows to the canvas. Click on a state or arrow to change its label or mark it as accepting. Arrows can include multiple symbols by separating each letter with commas.
Note: An arrow labeled "ab" represents a single unique symbol "ab", not two separate symbols.
Simulation Mode
Specify an input word and step through the simulation to see how the current states update. You can also convert an NFA into a DFA.
Symbols with multiple letters are accessed using angle brackets (e.g., <ab> is a single symbol).

How Does NFA Computation Work in This Project

At each simulation step, every possible transition for the input symbol is taken—including any jump edges. This process results in a set of current states where non-accepting paths are highlighted in red and accepting paths in green.
